# GitHub Copilot Chat Instructions

You are a code assistant specialized in Kintone plugin development with React and Redux.

## Project Context

- **Project Type**: Kintone Plugin (Kanban Board)
- **Frontend Framework**: React with Redux Toolkit
- **Build Tool**: Vite
- **Styling**: Tailwind CSS + Custom CSS modules
- **Target Environment**: Kintone platform (desktop and config)

## Coding Standards (CRITICAL - ALWAYS FOLLOW)

### Naming Conventions

- **JavaScript/TypeScript**: camelCase for variables, functions, methods
- **HTML**: kebab-case for class names, snake_case for id names
- **CSS**: kebab-case for class names (following BEM methodology)
- **File conversions**: Only convert hyphens ↔ underscores (no other changes)

### JavaScript Style Guide

- **Quotes**: ALWAYS use single quotes (') for JavaScript strings
- **HTML attributes**: ALWAYS use double quotes (") for HTML attributes
- **No ternary operators**: Use explicit if-else statements instead
- **Null/undefined checks**: Always check `!== null && !== undefined`

### Error Handling (MANDATORY)

- **Always use try-catch for**:
  - External API calls
  - User input processing
  - Async operations
  - JSON parsing
  - File operations
- **Specify error types** in catch blocks (SyntaxError, TypeError, etc.)
- **Never use empty catch blocks**

### JSDoc Comments (REQUIRED)

Every function must have JSDoc with:

```javascript
/**
 * @description [Function purpose]
 * @param {type} paramName - [Parameter description]
 * @returns {type} [Return value description]
 * @author nagasawa
 * @date 2025-06-09
 */
```

### Code Structure

- **React components**: Must use React hooks and Redux
- **Index files**: Create index.js in every folder for exports

### CSS Style Management (Recommended Guidelines)

- **Primary approach**: Use external CSS files for maintainability
- **Alternative options**: CSS-in-JS libraries and inline styles are acceptable when efficient
- **CSS file organization**:
  - Config components: `src/css/config/ComponentName.css`
  - Desktop components: `src/css/desktop/ComponentName.css`
  - Utility styles: `src/css/utilities/utility-name.css`
- **CSS file naming**: Should match component name when possible
- **CSS class naming**: Use kebab-case with BEM methodology preferred
- **Library usage**: Encourage use of styling libraries when they improve efficiency

### CSS Priority Order

1. **Preferred**: External CSS files (for consistency and maintainability)
2. **Acceptable**: CSS-in-JS libraries (styled-components, emotion, etc.)
3. **Acceptable**: Inline styles (for dynamic values or quick adjustments)

### CSS Examples

## Example Code Pattern

```javascript
// Component with external CSS (preferred)
import React, { useState } from 'react';
import './../../css/desktop/KanbanCard.css';

/**
 * @description カンバンカードコンポーネント
 * @param {object} props - コンポーネントのプロパティ
 * @param {object} props.record - レコードデータ
 * @param {string} props.customColor - カスタム背景色（オプション）
 * @returns {JSX.Element} カンバンカードコンポーネント
 * @author nagasawa
 * @date 2025-06-17
 */
const KanbanCard = ({ record, customColor }) => {
  // カードの状態管理（useStateで管理）
  const [isExpanded, setIsExpanded] = useState(false);

  if (record !== null && record !== undefined) {
    // レコードが存在する場合の処理
    const cardStyle = customColor ? { backgroundColor: customColor } : {};

    return (
      <div className='kanban-card' style={cardStyle}>
        <div className='kanban-card__header'>
          <h3 className='kanban-card__title'>{record.title}</h3>
        </div>
      </div>
    );
  } else {
    // レコードが存在しない場合の処理
    return <div className='kanban-card kanban-card--empty'>No data</div>;
  }
};

export default KanbanCard;
```

Remember: Quality, maintainability, and consistency are prioritized over speed.
